# 그룹화·레이아웃 기준 (design-dna)

> 이 문서는 **"무엇을 어떻게 묶고, 어떤 모양으로 담느냐"** 하나만 다룬다.
> 색·타이포·카피·사진 처리는 각각의 문서로 라우팅한다. 여기서는 **구조(오토레이아웃·계층·그룹화)** 와 **조형 처리 원리(왜 이 모양인가)** 만 규정한다.
> "어떻게(API 호출·함정 회피)"는 `figma-mcp.md`로 링크만 건다.

---

## 0. 대전제 — 왜 오토레이아웃인가

레이아웃에는 두 패러다임이 있다.

| 패러다임 | 방식 | 지위 | 왜 |
|---|---|---|---|
| 절대좌표형 | 모든 요소를 `absolute top-[N]`로 흩뿌림 | **레거시** | 요소 하나 추가/삭제하면 아래 전부 손으로 밀어야 하고, 카피가 바뀌면 붕괴한다 → 재사용이 어렵다 |
| 오토레이아웃형 | `flex column/row + gap` 스택 | **표준(확정)** | 자식이 늘면 부모가 스스로 늘어남(hug). 카피·개수가 바뀌어도 리듬 유지 |

**원칙: 새로 만드는 모든 것은 오토레이아웃형이다.** 절대좌표는 기존 레거시 프레임을 그대로 복제할 때만 허용한다.

**주의 — 실측 원본들은 절대좌표 레거시가 다수다.** 기존 파일(브랜드 불문)의 원본 프레임은 대부분 절대좌표로 만들어져 있고, 오토레이아웃은 일부 컴포넌트 내부에만 있다. 원본을 참조·복제해서 새로 만들 땐 좌표 관계를 오토레이아웃 제약으로 **번역**한다: 중앙정렬 `x=(W−w)/2` → `counterAxisAlignItems:'CENTER'`, 등간격 y 반복 → `itemSpacing`, 고정 컬럼폭 → 자식 `FIXED` 폭. 원본이 절대좌표라는 사실을 "절대좌표로 만들어도 된다"의 근거로 삼지 않는다.

## 1. 중첩 계층 기본형

레이아웃은 **3단 중첩**을 기본형으로 삼는다. 위에서 아래로 관심사가 좁아진다.

```
섹션 (column)              ← 세로로 쌓이는 큰 덩어리, 화면 하나
  └ 의미단위 카드/서브프레임 (column)   ← "한 가지 이야기" 단위 (카드 1장, 리뷰 1개)
       └ row               ← 나란히 놓이는 최소 조합 (아이콘+텍스트, 라벨+값)
```

| 계층 | 방향 | 담는 것 | 왜 이 방향 |
|---|---|---|---|
| 섹션 | column | 제목 블록 + 본문 블록 + 카드 스택 | 상세페이지는 위→아래로 스크롤. 큰 흐름은 세로 |
| 의미단위 | column | 사진 + 제목 + 설명 | 한 카드 안에서도 읽는 순서가 위→아래 |
| row | row | 아이콘+라벨, 라벨+값, 넘버+제목 | "함께 읽혀야 하는" 짝은 가로로 나란히 |

**분기 — 언제 row로 내려가나:** 두 요소가 *같은 순간에 한 눈에* 읽혀야 하면 row(아이콘+텍스트, 별점+날짜, 라벨+값). *순서대로* 읽혀야 하면 column(제목 다음 본문). 이 구분이 곧 방향 선택 기준이다.

### 텍스트 컨테이너 사이징 — hug냐 fixed냐는 카피의 역할이 정한다

텍스트를 감싸는 박스는 두 케이스뿐이며, **배치 전에 기획 원문의 뉘앙스로 역할을 판정해 사이징을 정한다**(만들고 나서 고치는 게 아니다).

| 케이스 | 사이징 | 왜 |
|---|---|---|
| **타이틀·강조문** — 섹션 제목 배지(pill), 카테고리 라벨, 제품 소구 한 줄 | **hug width** (폭 = 텍스트 + 좌우 패딩) | 1줄 선언문을 고정폭에 넣으면 텍스트 길이에 따라 좌우가 타이트하거나 헐렁해진다(실패 사례: 고정 300px 배지에 제목 텍스트가 좌우 꽉 낌). 배지는 내용물에 옷을 맞춘다 |
| **본문·설명** — 2문장 이상, 읽는 텍스트 | **fixed width + hug height** (폭 고정 — §7 최대폭 이내, 높이는 줄 수 따라 hug) | 읽는 문단은 폭이 고정돼야 가독폭·정렬 리듬이 유지되고, 카피가 늘면 아래로 자란다 |

애매하면 "이 카피가 2줄로 늘어날 수 있는 종류인가?"를 물어라 — 늘어날 수 있으면 fixed width다. (Figma 구현: hug width = `textAutoResize='WIDTH_AND_HEIGHT'` + 컨테이너 hug(AUTO) / fixed width+hug height = `textAutoResize='HEIGHT'` + FIXED 폭 — 순서 함정은 `figma-mcp.md`·`troubleshooting.md` T1·T3)

## 2. 의미단위 그룹화 — leaf 흩뿌리기 금지

**철칙: 의미상 한 덩어리인 것은 반드시 하나의 오토레이아웃 프레임으로 묶는다. leaf(텍스트·이미지·벡터)를 섹션 최상위에 직접 흩뿌리지 않는다.**

반드시 묶어야 하는 대표 짝:

| 의미단위 | 묶는 이유(왜) |
|---|---|
| 이미지 + 캡션 | 캡션은 그 이미지를 설명. 떨어지면 어느 사진 설명인지 모름 → 함께 이동·정렬돼야 함 |
| 강조어 + 배경(말풍선/배지) | 배경은 그 단어를 강조하려 존재. 분리되면 배경만 떠다님 |
| 아이콘 + 라벨 | 아이콘은 라벨의 시각 보조. 한 쌍으로 읽힘 |
| 넘버 + 제목 + 하이라이트 | 하나의 "장점 항목". 번호만 따로 놀면 안 됨 |
| 라벨 + 값 (성분표 행) | 좌우 2컬럼이 한 행. 행 단위로 반복·정렬 |

**왜 묶나:** ① 묶인 단위는 통째로 이동·복제·정렬된다(리듬 유지). ② 부모가 자식을 hug하므로 카피가 길어져도 레이아웃이 안 깨진다. ③ 흩뿌린 leaf는 요소 하나 추가할 때마다 전부 손봐야 한다.

**분기 — 묶을지 말지 헷갈릴 때:** "이 둘을 따로 다른 위치로 옮길 일이 있나?"를 물어라. 없으면 묶는다. 있으면 각각 별도 단위다.

## 2b. 반복·동일소속 요소는 한 단계 더 묶는다 + 그룹 간 ≥50px

§2가 "의미상 한 덩어리(카드 1장)"를 묶는 규칙이라면, 이건 그 **위 계층**이다. **같은 디자인 요소를 쓰는 반복 카드(리뷰카드 1~N, 스펙·티어·기능 카드), 또는 소속이 같은 하위 블록들은 한 번 더 상위 개념 그룹으로 묶는다.** 카드 각각을 묶는 데 그치지 말고, 같은 종류 N개를 하나의 상위 오토레이아웃 프레임으로 감싼다.

| 상황 | 처리 |
|---|---|
| 동일 규격 반복 카드(리뷰 5장·티어 3장·기능 4장) | 카드들을 감싼 상위 그룹 1개(`리뷰_N선`·`티어_묶음` 등, 배경 없는 정렬 래퍼 §5) |
| 소속이 같은 하위 블록(한 챕터의 카드들) | 챕터 단위 상위 그룹 |
| 성격이 다른 인접 블록(헤더 ↔ 카드묶음 ↔ 백업/부록) | 각각 별도 그룹, 사이는 아래 간격 규칙 |

**그룹 간 간격 규칙: 개념적으로 다른 두 그룹 사이는 최소 50px — 그룹 내부 간격보다 확연히 크게(권장 내부간격의 2배+).**

- **왜 한 번 더 묶나:** 같은 소속끼리 상위 그룹으로 감싸면 "이건 한 세트"라는 시각 위계가 생긴다. 안 묶으면 반복 카드와 그 위/아래 헤더·부록이 전부 같은 레벨 형제로 평평히 나열돼 어디까지가 한 묶음인지 안 읽힌다(반복 카드가 헤더·부록과 같은 레벨·좁은 간격으로 나열되면 전형적 실패).
- **왜 ≥50px:** 그룹 경계 간격이 내부 카드 간격(예: 16px)과 비슷하면 경계가 사라져 전부 한 덩어리로 뭉갠다. 그룹 사이를 내부보다 확연히 벌려야(≥50px) "여기서 다른 영역이 시작"이 선언된다. 다른 영역이 타이트하게 붙으면 그룹감이 죽는다.
- 구현: 반복 카드 상위 그룹은 `layoutMode=VERTICAL`+내부 `itemSpacing`(카드 간격) 유지, 상위 컬럼의 그룹 사이는 `itemSpacing≥50` 또는 그룹 프레임 상/하 `padding`으로. (reparent 후 layoutAlign 리셋 함정은 `troubleshooting.md` T15.)

## 3. 조형 처리 원리 — "왜 이 모양인가"

같은 내용도 담는 그릇의 모양이 역할을 말한다. **모양은 장식이 아니라 신호다.**

| 조형 처리 | 언제 쓰나(역할·상황) | 왜 이 모양 |
|---|---|---|
| **면채움 (solid fill)** | 강한 시각 블록이 필요할 때 — 카드 본체, 버튼, 말풍선 | 존재감·클릭 유도. "여기 독립된 덩어리가 있다"를 채운 면으로 선언 |
| **라인 테두리 (stroke)** | 내용을 담는 "그릇"을 은은히 정의할 때 — 카드 경계, 정보박스 | 배경과 대비가 약할 때 경계만 그어 담는다. 면을 채우면 과함 → 선으로 최소 정의 |
| **라운딩 (rounded)** | 카드·말풍선·버튼 대부분 | 브랜드 톤이 부드럽고 친근 → 둥근 모서리. 각진 처리는 신문형(레거시)에서만 |
| **얇은 divider vector** | 구획/구분 — 넘버 항목 사이, 성분표 행 사이. **N개 항목엔 N−1개 구분선**(마지막 항목 뒤엔 없음) | "나누되 무게는 싣지 않는다." 채운 도형이 아니라 얇은 선으로 구분만 |
| **밑줄 vector** | 특정 단어 강조 | 단어 아래 선으로 포인트. 단 카피가 바뀌면 위치 재계산 필요(텍스트에 자동 종속 아님) |
| **배경 없는 대형 텍스트 + 아이콘** | 강조 헤드라인·CTA — **우아·감성 톤 브랜드** | 크기·굵기 점프 + 아이콘만으로 강조. 박스를 두르면 "버튼"처럼 축소돼 보이는 톤에서 채택 |
| **배지형 타이틀/CTA 앵커 (pill)** | 섹션 제목·CTA — **팝·스티커 톤 브랜드** | 흰 면 + 굵은 컬러 보더 + 라운딩 + (하드 오프셋) 섀도의 알약 배지로 제목을 감싼다. **보더/텍스트 색만 스와핑**해 같은 배지가 중립/경고/제품라인 톤을 오간다 — 규격은 고정, 색이 의미를 나른다 |

### 상황 분기 — 같은 "강조"라도

- **독립된 샘플을 격리해 보여줄 때** → 테두리 + 라운딩 + (연한) 면채움 배경을 다 갖춘 카드. (사진 뒤 연한 면 = 안정감, 얇은 라인 = 경계 정의, 라운딩 = 톤.) 하나의 완결된 카드로 격리.
- **문구를 세게 밀어붙일 때(CTA·섹션 타이틀)** → **브랜드 톤이 결정한다.** 우아·감성 톤 = 박스 없는 대형 텍스트(+아이콘). 팝·스티커 톤 = 배지형 앵커(pill). 어느 쪽인지는 그 브랜드 프로파일(`brands/<이름>.md`)의 톤 선언을 따르고, 한 페이지 안에서는 한 방식으로 통일한다.
- **정보를 정렬해 보여줄 때(고시/구성표)** → 라운드 테두리 박스 + 행 사이 얇은 구분선. 각 값에 면을 채우지 않는다(정보는 조용해야 함).
- **구조를 구획할 때(번호형)** → 채운 도형이 아니라 얇은 선(divider/밑줄). 구획은 나눔이지 블록이 아니다.

### 반복 섹션의 모듈 설계 — 골격 고정 + 미디어 슬롯 가변

기능 상세/차별점처럼 같은 유형 섹션을 N개 반복할 때: **상단 배지·부제·본문의 골격과 프레임 폭은 고정**하고, **이미지 존만 변주**한다(단일/2분할/3콜라주/겹침 등 — 높이는 콘텐츠에 비례해 연장). 왜: 골격 고정이 "같은 종류"라는 리듬을 만들고, 미디어 변주가 지루함을 깬다 — 반복 리듬과 콘텐츠 다양성을 양립시키는 표준 해법. §4의 "의도적 가변 예외"가 바로 이 모듈이다.

### 말풍선의 특수성
말풍선의 정체성은 **꼬리**다. 둥근 몸통 + 45° 꼬리를 하나의 도형으로 합쳐야 "말풍선"이 된다. 몸통만 있으면 그냥 알약 배지다. (합치는 과정의 fill 리셋 등 API 함정은 `figma-mcp.md`·`troubleshooting.md` 참조.)

## 4. 사이즈 통일 = 리듬

반복되는 요소는 **크기와 간격을 통일**해 리듬을 만든다.

| 규칙 | 기준 | 왜 |
|---|---|---|
| 반복 카드는 같은 크기 | 카드 스택 안 모든 카드 동일 치수 | 통일하면 시선이 규칙적으로 흘러 리듬이 생기고, 들쭉날쭉하면 정렬 리듬이 깨진다 |
| gap은 고정 | 카드 사이 간격을 상수로 | 간격이 일정해야 시선이 규칙적으로 흐름 |
| 개수 늘면 height가 hug로 늘어남 | 항목 추가 시 gap을 줄여 욱여넣지 **않음** | 간격을 희생하면 리듬이 깨짐. 부모가 세로로 자라게 둔다 |

**분기 — 통일이 규칙이 아닌 경우:** 원근감·서사 연출을 위해 **의도적으로** 크기를 가변으로 둘 때가 있다(예: 원근 카드 배치). 이건 "리듬을 깬 게 아니라 다른 리듬(깊이)을 만든 것". 의도가 있으면 예외, 없으면 통일이 기본이다.

## 5. 정렬 래퍼 함정 — 투명 배경

순수 정렬·간격만을 위한 래퍼 프레임에 면이 깔려 있으면, 그 면이 콘텐츠 위를 덮어 의도치 않은 흰 박스가 겹쳐 보인다.

**규칙: 정렬·간격 전용 래퍼는 배경을 비운다.** 배경이 필요한 카드/블록만 면을 채운다.
**왜:** 래퍼의 역할은 "자식을 정렬"이지 "면을 보여줌"이 아니다. 역할이 정렬이지 면 표시가 아니므로, 역할에 없는 면은 제거한다. (오토레이아웃 프레임 생성 시 기본으로 면이 깔리는 메커니즘과 배경을 비우는 구체적 방법은 `figma-mcp.md`·`troubleshooting.md` T8 참조.)

## 6. 네이밍 — 역할 한글명

**규칙: 모든 노드에 역할을 드러내는 한글명을 붙인다. `Frame 12` 같은 기본명 금지.**

| 좋은 예 | 나쁜 예 |
|---|---|
| `사진`, `제목`, `본문`, `구분선`, `말풍선_꼬리`, `개성카드`, `리뷰박스` | `Frame 12`, `Group 3`, `Rectangle 5` |

**왜:** ① 나중에 찾고 수정할 때 역할이 곧 좌표다. ② 재사용·복제 시 무엇을 복제하는지 즉시 안다. ③ 기본명은 계층이 깊어질수록 미로가 된다.

**실측 반면교사** (기존 원본에서 실제로 만연했던 부패 유형 — 새 작업에서 금지):
- 텍스트 레이어가 전부 기본명(첫 글자 자동명 등) — 수십 개가 구분 불가.
- 작업 메모가 레이어명에 잔존("회색빼다", "진짜박스") — 지시가 아니라 이름이어야 한다.
- 복붙 후 미개명: 다른 제품의 이름이 새 제품 프레임에 그대로 남음(A제품 라벨이 B제품 페이지에), 동명이물 컴포넌트(같은 이름의 전혀 다른 두 배지).
- 이미지 레이어가 타임스탬프 파일명 그대로.

**좋은 관행으로 흡수**: 최상위 섹션 프레임의 **넘버링 네이밍**(`1. 오프닝페이지`, `2-1. 시스템 소개`, `기능 상세 설명 3`) — 이름이 곧 페이지 정보 구조가 되어 순서·소속이 즉시 읽힌다. 새 작업의 최상위 프레임에 채택.

## 7. 규격 원리 — 절대값은 브랜드/플랫폼 종속, 원리만 범용

규격에서 **범용인 것은 원리**이고, **구체 수치는 브랜드·플랫폼 값**이다(→ `brands/<이름>.md`). 단 아트보드 폭은 교차 검증 결과 기본값이 확립됐다(아래).

| 규격 원리 | 기준 | 왜 |
|---|---|---|
| **상세페이지 아트보드 폭 = 860px 기본값** | 여러 브랜드(식품/디지털)·전 시기·수십 프레임에서 예외 없이 860 단일값 확인 → **기본값(default)으로 승격.** 플랫폼 규격이 다르면 그 값을 따르되, 별도 지시 없으면 860 | 전 프레임 폭이 동일해야 섹션들이 정렬·연속되고, 이 폭이 가독폭 계산의 기준선이 된다. 세로는 콘텐츠 분량에 따라 가변(표준 높이 없음) |
| **상세페이지 폭 ≠ 파생 채널 폭** | SNS 피드(예: 1080×1350), E북/PDF(595×842 A4), 앱 목업(375) 등은 **별개 규격** — 상세페이지 폭을 따르지 않는다 | 파생물은 각 채널의 표준 캔버스가 있다. 상세페이지에서 파생 채널로 크롭할 땐 채널 규격으로 새 프레임을 뜬다 |
| **다중 컨테이너 폭 체계 + 텍스트 최대폭 하드 리밋** | 본문 텍스트 컬럼(≈폭의 80%, 860→~691) < 텍스트가 든 카드·박스·필 ≤ **캔버스폭의 84%(하드 리밋, 860→725)** — 관용 운용대 78~84%(860→675~725). 이 한계를 넘어 full bleed까지 가는 것은 **순수 시각물(사진·목업·색 밴드)뿐** | 텍스트 콘텐츠가 84%를 넘으면 좌우 여백이 죽어 화면이 답답해진다(실패 사례: 텍스트 카드 780@860=91%. 원본 실측 한계 725). 시각물만 넓게 나가야 "여백↔존재감" 리듬이 산다 |
| **가독폭 = 아트보드폭 − 좌우 여백** | 본문 한 줄이 이 폭을 넘지 않게. 여백 비율은 대략 폭의 8~10%씩 | 한 줄이 너무 길면 눈이 다음 줄을 못 찾는다. 넘으면 카피를 줄인다(폰트 축소 금지 → `04-copy`) |
| **반복 카드/컴포넌트는 한 규격으로 통일** | 같은 스택 안 모든 인스턴스 동일 치수 | 통일이 리듬·"같은 종류"라는 신호를 만든다. 구체 치수는 콘텐츠에 맞춰 정하되 스택 안에서는 불변 |
| **라운딩은 브랜드 톤의 표준값 하나로** | 카드·사진에 같은 모서리값 반복 | 모서리값이 섞이면 톤이 흔들린다. 값 자체는 브랜드 프로파일 |
| **말풍선 라운딩 ≈ 높이의 42%** | 비례값(절대 아님) | 알약형 몸통은 높이 대비 비율로 잡아야 크기가 달라져도 형태 유지 |

**규격의 구체 치수(아트보드 폭 외의 카드 크기·리뷰박스·라운딩값 등)는 전부 브랜드 프로파일에서 확인**한다. 프로파일에 없으면 유저에게 묻는다 — 패턴만으로 절대 치수를 지어내지 않는다.

### 7b. 프레임 하단 클리어런스 — 최하단 100px 비우기

**규칙: 어떤 텍스트·조형요소도 그 프레임(섹션/페이지)의 하단 경계 100px 이내에 두지 않는다.** 섹션 프레임 하단 여백(paddingBottom) ≥100px, 절대배치 히어로는 최하단 요소의 바닥 ≤ (프레임 높이−100).

- **왜:** 요소가 프레임 바닥에 바짝 붙으면 잘린 듯 답답하고, 아래 섹션과 시각적으로 뭉개져 경계가 사라진다. 하단 100px 여백이 "이 섹션은 여기서 끝난다"는 숨을 준다. 페이지 최하단(마지막 섹션 바닥)도 동일(요소가 바닥 수십 px에 붙으면 전형적 실패).
- 상단도 대칭 여백을 권장하되, **하단 붙음이 더 흔한 실패**라 하단을 하한으로 못박는다.
