# design-dna / 카피 — 목적·길이 도해

> 이 문서는 **오직 "카피(문안)의 역할과 길이"** 하나만 다룬다. 글자의 크기·굵기·색은 03-typography, 배치·묶음은 02-grouping-layout, 페이지별 목적·첫인상 효과는 05-page-purpose로 라우팅한다. 여기서는 "어떤 역할의 카피가 무엇을 달성하려 하며, 얼마나 길어도 되는가"만 결정한다.

---

## 0. 대전제 — 길이는 글자수가 아니라 "폭·역할"로 판단한다

카피 길이를 "몇 글자 이내"로 세지 마라. 두 가지 축으로만 판단한다.

1. **역할 축**: 이 문안이 첫인상(디스플레이)인가, 읽어야 할 설명(본문)인가, 훑는 라벨(스펙)인가. 역할이 허용 길이를 정한다.
2. **폭 축**: 실제 렌더된 한 줄이 **가독폭**을 넘느냐. 본문 가독폭은 아트보드 폭에서 좌우 여백을 뺀 값이다(관용값은 브랜드 프로파일 참조 — 예: 860폭 기준 ~691px). 이 폭 안에서 눈이 다음 줄로 편하게 넘어간다.

**왜 글자수가 아닌가**: 같은 10글자도 디스플레이 사이즈면 화면을 꽉 채우고, 캡션 사이즈면 한 줄의 1/4이다. 길이의 실제 문제는 "한 줄이 가독폭을 넘어 읽기 불편해지는 것"이지 글자 개수가 아니다. 그래서 판정 기준은 항상 **렌더 폭 대비 역할**이다.

**핵심 분기 — 넘칠 때 무엇을 하는가**:

| 상황 | 하지 말 것 | 해야 할 것 | 왜 |
|---|---|---|---|
| 제목/헤드라인이 가독폭을 넘음 | 폰트를 줄여 억지로 한 줄에 욱여넣기, 임의로 단어 잘라 축약 | **"카피를 줄여달라"고 되묻는다** | 디스플레이는 크기로 압도하는 게 역할이다. 크기를 희생하면 역할이 죽는다. 문안 자체가 길다는 신호이므로 문안을 고쳐야 한다 |
| 본문 한 줄이 가독폭을 넘음 | 폰트 축소 | 문단을 더 짧게 끊고 **문단 사이 여백**으로 호흡 | 본문은 크기가 아니라 가독성이 역할. 줄바꿈·여백으로 해결 |
| 스펙 값이 셀을 넘음 | 라벨 컬럼까지 침범 | 값을 2줄로 접거나 표현을 압축 | 라벨-값 정렬(리듬)이 스펙의 역할 |

## 1. 역할별 카피 도해 (역할 → 길이 기준 → 왜)

| 역할 | 길이 기준 | 왜 이 길이인가 |
|---|---|---|
| **제품명(디스플레이)** | 짧은 **명사구** 1줄. 절대 줄바꿈 없음(nowrap) | 디스플레이는 크기로 압도하는 역할이라, 크게 키워도 한눈에 읽히는 짧은 명사구 1줄이어야 한다. 길거나 줄바꿈되면 크기를 못 키워 역할이 죽는다. (첫인상 목적 지표는 05 참조) |
| **헤드라인(사회적 증거·단언)** | 짧은 **사회적 증거 단언** 1줄 (완판·리뷰점수 유형의 지표) | 신뢰지표는 길면 힘이 빠진다. 한 방으로 꽂아야 강도가 산다 → 1줄 단언 |
| **질문형 후킹(페인포인트)** | **3줄 병렬**, 각 1줄, 같은 어미 반복 | 페인포인트를 리듬으로 두드려 공감을 쌓는다. 병렬·반복 어미가 "내 얘기다" 감을 만든다 → 그래서 3줄 세트 |
| **개성 카드 제목(후킹)** | 한 줄 후킹(질문/단언), 길면 2줄 | 카드 하나의 강점을 한 호흡에. 훅이므로 짧아야 다음 본문으로 끌고 간다 |
| **본문/설명** | **훅 1줄 + 설명 2~3줄**, 각 줄 가독폭 이내, 문단 사이 여백 | 훅으로 붙잡고 설명으로 납득시킨다. 2~3줄이 한 화면에서 부담 없이 읽히는 분량 |
| **리스트/스펙(라벨-값)** | 라벨·값 각각 **짧게**, 2컬럼, 넘버링은 `01~` | 훑어 읽는 정보. 라벨-값 2컬럼 정렬이 스캔을 빠르게 한다. 넘버링은 순서·완결감을 준다 |
| **말풍선 인용** | **짧은 구어체 1줄**, 진짜 같은 불만 톤 | 실제 후기처럼 들려야 신뢰가 산다. 길면 대사가 아니라 글이 된다 → 1줄 구어체 |
| **CTA(전환)** | **초강조 짧은 명령형 구절** | 행동을 촉구하는 마지막 방아쇠. 짧을수록 방아쇠가 강하다 |
| **법정 고지문** | **정해진 정확 문구 그대로**, 수정·축약 금지 | 법적 표기라 문안을 바꾸면 위반. 길이 규칙 자체가 적용되지 않는 예외 |

## 2. 상황별 분기 ("이럴 땐 이렇게")

### 2-1. 제목이 길어질 때
- **원칙**: 디스플레이 카피는 폭을 넘기면 문안을 줄인다. 크기·자간을 깎아 맞추지 않는다.
- **왜**: 제목의 임무는 "크게 압도"이지 "정보 다 담기"가 아니다. 정보는 아래 본문이 맡는다. 크기를 지키고 문안을 되묻는 게 역할 분업에 맞다.
- 실무 행동: 제작자가 임의로 자르지 말고, **카피 담당에게 "이 자리는 1줄 명사구/단언이어야 한다, 줄여달라"고 요청**한다.

### 2-2. 페인포인트가 2개거나 5개일 때
- 3줄 병렬은 관용 리듬이지 마법의 숫자가 아니다. 다만 **병렬·같은 어미·각 1줄** 원칙은 유지한다.
- **왜**: 후킹의 힘은 개수가 아니라 "같은 형식의 반복"에서 온다. 개수가 바뀌어도 반복 구문 구조를 지키면 리듬이 산다.

### 2-3. 설명이 길어서 3줄을 넘을 때
- 한 덩어리로 늘리지 말고 **문단을 나눠 문단 사이 여백**을 준다.
- **왜**: 긴 텍스트벽은 읽기를 포기하게 한다. 짧은 문단 + 여백이 "읽어도 되겠다"는 호흡을 만든다. 길이 문제를 크기가 아니라 구조로 푼다.

### 2-4. 말풍선 후기를 여러 개 놓을 때
- 각 말풍선은 **짧은 구어체 1줄**이어야 대화감이 산다. 문안이 길면 대사가 아니라 글이 되어 현장감이 죽는다.
- 톤: 매끈한 카피가 아니라 **진짜 같은 불만/구어체**. 다듬을수록 광고처럼 들려 신뢰가 떨어진다.
- 배치(좌우 교차 등 대화감을 만드는 정렬)는 이 문서 소관이 아니다 → **02-grouping-layout 참조**.

### 2-5. 스펙/성분표를 채울 때
- 라벨-값 2컬럼, 값은 짧게. 넘버링이 필요한 목록은 `01`부터.
- **법정 고지문이 섞이면** 그 부분은 **문안 자체를 수정·축약하지 않는다(정확 문구 고정)**.
- **왜**: 스펙은 신뢰·투명성이 역할이라 라벨-값 정렬(스캔성)과 문구 정확성이 길이보다 우선한다. 고지문은 정확성이 절대 우선이라 카피 규칙의 예외다.
- 고지문이 폭을 넘칠 때의 레이아웃 처리(폭·행 배분 등)는 카피 소관이 아니다 → **02-grouping-layout 참조**.

### 2-6. 이모지 사용 (친근 톤 브랜드)
- **아이콘 대체 용법**: 커스텀 아이콘 제작 없이 **대형 이모지**(디스플레이급 크기)를 카드의 감정 반응·페르소나 라벨(직업 이모지+텍스트 필)로 쓴다. 왜: 이모지는 제작비 0으로 감정·카테고리를 즉시 전달하고, 톤이 친근한 브랜드에선 오히려 커스텀 아이콘보다 자연스럽다.
- **문장 말미 톤 용법**: 본문 문장 끝에 이모지를 붙여 구어체 친근 톤을 유지한다. 페이지 전체에서 쓰는 방식(붙인다/안 붙인다)을 통일 — 섞이면 톤이 흔들린다.
- **분기**: 진중·감성 톤 브랜드/섹션(철학 등)에서는 쓰지 않는다. 이모지는 친근 톤의 장치이지 기본값이 아니다. 브랜드 톤 선언(brands/) 참조.

## 3. 서사형(설득) 카피의 길이 흐름

문제제기 → 원인진단 → 리빌 → CTA로 이어지는 설득 구조에서도 각 단계의 길이 역할은 다음과 같다.

| 단계 | 카피 길이 역할 | 왜 |
|---|---|---|
| 헤드라인(품질 단언) | 짧은 단언 1줄 | 서사의 문을 세게 연다 |
| 불만 후기(말풍선) | 짧은 구어체 1줄 × 여러 개 | 공감·현장감. 길면 대사감이 죽는다 (배치는 02 참조) |
| 원인 진단 | 1문장 | 전환점은 짧고 명료해야 각인된다 |
| 리빌("진짜 이유는 ~") + CTA | 리빌 1~2줄 + 초강조 명령형 CTA 구절 | 납득 후 즉시 행동 유도. CTA는 짧을수록 방아쇠가 강함 |

**왜 이 흐름인가**: 길이가 감정 곡선을 만든다. 짧게 때리고(후기) → 짧게 짚고(진단) → 풀어주고(리빌) → 짧게 밀어낸다(CTA). 어느 단계든 문안이 길어지면 그 단계의 감정 기능이 무뎌진다.

## 4. 판정 체크리스트 (새 제품 카피를 앉힐 때)

1. 이 자리의 **역할**은? (디스플레이/헤드라인/후킹/본문/스펙/인용/고지/CTA)
2. 그 역할의 **허용 길이**는? (1줄 명사구·단언 / 3줄 병렬 / 훅1+설명2~3 / 라벨-값 / 1줄 구어체 / 초강조 명령형 구절 / 정확문구고정)
3. 렌더된 한 줄이 **가독폭**(브랜드 프로파일 값)을 넘는가?
4. 넘친다면 — **디스플레이는 문안을 줄이라고 되묻고**, 본문은 문단을 끊고, 스펙은 값을 접는다. **폰트를 깎아 욱여넣지 않는다.**
5. 법정 고지문은 위 규칙에서 제외 — **원문 그대로**.

---

> 판정은 폭·역할로만 한다 — 절대 글자수 기준은 두지 않는다. 특정 제품의 실제 문안·배치·페이지 목적 지표는 이 문서 소관 밖(인스턴스 데이터 또는 02·05 소관)이라 역할·길이 규칙으로만 서술했다.
