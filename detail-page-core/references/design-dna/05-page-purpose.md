# 페이지별 목적 — 섹션 타입 라우팅

> 이 문서는 **"어떤 섹션을, 왜, 무엇을 달성하려고 쓰는가"** 하나만 다룬다.
> 색·타이포·레이아웃·카피 길이·이미지 처리의 *구체 규칙*은 각 전용 문서(01~04, 06)로 라우팅한다. 여기서는 각 섹션의 **역할·필수요소·톤·선택조건**만 정의한다.
> 핵심 원리: **페이지는 "역할"로 고르고, "보유자산(후기 수·인증 유무·성분 데이터 등)"으로 비중을 조절한다.**

---

## 0. 이 문서를 쓰는 법 (원리 우선)

상세페이지는 "예쁜 화면의 나열"이 아니라 **구매 결정을 한 계단씩 밀어 올리는 설득 시퀀스**다. 각 섹션은 그 계단 중 하나의 심리적 과업을 맡는다. 그래서 섹션을 고를 때 던질 질문은 "무슨 디자인?"이 아니라 **"이 자리에서 고객의 어떤 마음을 움직여야 하는가?"** 다.

| 판단축 | 기준 | 왜 |
|---|---|---|
| 섹션 선택 | 달성할 **심리 과업**(첫인상/공감/근거/차별화/신뢰/투명성/정서)으로 고른다 | 같은 제품도 과업이 다르면 다른 섹션이 필요하다. 디자인 형태는 과업의 결과지 출발점이 아니다 |
| 섹션 비중 | **보유자산의 양**으로 조절한다(후기가 많을수록 리뷰 지면을 키우고, 인증이 많을수록 번호를 늘린다) | 없는 자산을 채우려 억지 섹션을 넣으면 신뢰가 오히려 깎인다. 강한 자산에 지면을 몰아줘야 설득이 집중된다 |
| 정서 섹션 | **새로 만들지 않고 기존 프레임을 복제**한다(철학·인스타 노하우) | 브랜드 공용 자산이라 제품마다 재디자인하면 브랜드 정체성이 흔들린다. 일관성이 곧 브랜드 신뢰 |

## 0.5 이미지 주인공이냐, 텍스트 주도냐 — 페이지 아키타입 (★세로 사이즈를 결정하는 최상위 판단)

> **이 판단을 스켈레톤 前에 먼저 한다.** 페이지의 세로 길이와 템포는 "텍스트를 다 쓴 뒤 남는 이미지 자리"가 아니라, **시각 히어로 구간을 먼저 예약하고 그 사이에 텍스트를 채우는 순서**로 결정된다. 순서를 뒤집으면(텍스트를 먼저 다 채우고 이미지를 남는 자리에 소품처럼 끼우면) 세로가 텍스트로만 가득 찬 페이지가 나온다 — 전형적 실패.

### 아키타입 2종
| 아키타입 | 설득의 축 | 텍스트 밀도 | 시각 히어로 빈도 | 예 |
|---|---|---|---|---|
| **감각 설득형 (이미지 주인공)** | 제품의 감각적 끌림(먹음직스러움·질감·발색·향·형태·사용장면·패키지) | 낮음 — 핵심어 위주 | 많음(3~5구간) — 제품컷이 주인공인 씬을 중간중간 배치 | 식품·뷰티·리빙 등 실물 상품 |
| **정보 설득형 (텍스트 주도)** | 기능·스펙·사용법·비교(눈으로 만져지지 않는 가치) | 높음 — 설명·비교가 본체 | 적음(1~2구간) — 오프닝·클로징 정도 | 앱·SaaS·디지털 템플릿·서비스·구독 |

### 판정 트리거 — 이 페이지가 이미지 주인공인가 (스켈레톤 착수 시)

**Q1. 실물성** — 손에 잡히는 실물 상품인가?
> **카테고리 앵커(관습상 큰 제품사진·연출컷이 기본)**: 음식·식품 / 가구·리빙 / 전자기기·가전 / 뷰티·화장품 / 패션·의류·잡화 / 반려·키즈 실물 / 문구·소품. → 이 카테고리면 Q1 YES.
> NO: 앱·SaaS·구독·디지털 템플릿·서비스·컨설팅 등 무형.

**Q2. 감각 의존도** — "감각적 매력 있나?"는 주관 판단이라 LLM이 쉽게 건너뛴다. **관찰 가능한 속성 체크리스트로** 판정하고, 아래 중 **하나라도** 해당하면 YES:
- 제품/원물 클로즈업이 **먹음직스러움·소장욕**을 부르는 시각 디테일(질감·결·윤기·물방울·김·단면·발색)을 가진다
- **색/발색** 자체가 셀링포인트다
- **형태·실루엣·패키지**가 눈에 띄어 소장·선물 욕구를 부른다
- **사용/취식 장면**(붓기·바르기·입기·얹기·꺼내기)이 그 자체로 매력적이다
- 경쟁사 상세페이지가 대개 **큰 제품사진·연출컷**을 쓴다(카테고리 관습 — 객관 신호)
- 스펙표·기능설명 없이 **사진만 봐도** 뭔지·왜 좋은지 전달된다
> ★ **애매하면 감각형으로 기운다.** 히어로 자리를 안 잡는 false negative(텍스트만 가득 찬 페이지)가 훨씬 손해다. 정보형으로 확정하는 건 "이 제품은 사진보다 **설명·비교·기능**이 팔리게 한다"가 분명할 때만.

**Q3. 비주얼 자산** — 스킬 착수 시점에 사진 인벤토리를 확인한다(LLM은 자기 입력을 보므로 판정 가능):
- 사용자가 **이미지 첨부**(제품컷·원물·사용컷·기존 상세페이지 캡처)를 했거나,
- **기존 상세페이지·브랜드 자산 선반**에서 크롭·복제 가능한 실사가 있거나,
- 없다면 **촬영/생성 계획**이 명시됐는가.
> Q3는 페이지 타입을 뒤집지 않는다 — 사진이 아직 없어도 감각형이면 슬롯을 예약하고 "촬영/생성/placeholder 필요"로 표시. Q3가 정하는 건 **히어로 자리를 무엇으로 채우나**(실사 / 배경제거 누끼 / 생성 / placeholder).

**판정: Q1·Q2·Q3 중 2개 이상 YES → 감각 설득형(이미지 주인공).** 그 외 → 정보 설득형.
> 배타적 스위치가 아니라 **밀도 다이얼**이다. 식품이라도 성분·근거 구간은 정보형으로 촘촘하고, 디지털이라도 오프닝 히어로는 시각형으로. 최종 확정은 인터뷰로(→ 00-interview).

### 공간 예약 원칙 (세로 사이즈 결정 순서) ★
스켈레톤 단계에서 텍스트를 채우기 **전에**:
1. **시각 히어로 구간을 먼저 지정**한다(우선순위는 §1-15).
2. 각 히어로 구간에 **넉넉한 세로 공간을 먼저 예약**한다 — 상·하한이 둘 다 있다:
   - **텍스트 점유 면적 = 섹션 면적(폭×높이)의 12~20%.** 상한 20%(이미지가 주인공 — 문장 늘리지 말 것) + **하한 12%(텅 빈 히어로 방지)**. 헤드라인·짧은 본문 없이 이미지만 덩그러니 두면 "공허하고 전달 내용 0"인 실패가 난다(텍스트 사실상 0%면 폰트도 작아 보여 최악). ★기준은 **세로 점유가 아니라 면적**(텍스트 블록들의 바운딩 면적 합 ÷ 섹션 면적, 글자수 아님) — 폭 좁은 텍스트가 세로로만 길면 세로기준 20%라도 화면은 휑하다.
   - **히어로 높이 = 일반 정보 섹션의 1.8~2.5배, 단 세로 최대 3000px(하드 상한).** 1.8~2.5배가 3000을 넘으면 3000으로 캡한다 — 3000 초과 히어로는 스크롤이 휑하고 폰트가 상대적으로 작아 보여 공허해진다. 전면 히어로 사진 높이 ≥ 캔버스폭×0.7, 부유컷은 제품 위아래 여백 각 100px+ — 조형은 06 문서.
   - 감각형 페이지 전체에서 히어로 **3~5구간**. **각 섹션 하단 100px·반복그룹 간 50px 클리어런스(→ 02 문서)**도 세로 예약 시 함께 반영.
3. **그 다음** 텍스트·정보 섹션을 그 사이 리듬에 채운다.
- 순서 금지: 텍스트를 먼저 다 채우고 이미지를 남는 자리에 끼우기 → 세로가 텍스트로만 차고 템포가 죽는다.

## 1. 섹션 카탈로그 — 역할 → 필수요소 → 톤 → 왜

각 섹션의 절대값·색·폰트 수치는 여기에 두지 않는다. "무엇을/왜"만 정의하고, "어떻게"는 전용 문서로.

### 1-1. 소개 메인 (인트로 / 후킹)

| 항목 | 내용 |
|---|---|
| **달성 과업** | 스크롤 3초 안에 "이게 무슨 제품이고, 왜 믿을 만한가"를 각인 |
| **필수요소** | ① 상단 헤드라인(사회적 증거 — 완판/리뷰점수 등 신뢰지표) ② 중앙 제품 누끼(주인공) ③ 하단 제품명 3단(작은 서브캡션 → 큰 제품명 → 태그라인) |
| **톤** | 첫인상, 압도적. 화면을 가득 채우는 존재감 |
| **왜** | 첫 화면에서 **컨셉과 신뢰를 3초 안에 동시에** 못 박아야 고객이 다음 스크롤로 넘어간다 → 여기서 둘 중 하나라도 놓치면 아래 섹션은 읽히지 않는다. 제품명을 페이지의 주인공으로 세우는 건 "이 페이지가 무엇에 관한 것인지"를 시선 위계로 선언하는 것(→ 크기·서체 위계는 03 문서) |

> 신뢰지표가 없는 신제품이라면? → 헤드라인을 "완판/점수" 대신 **제품 컨셉 단언**(핵심 강점 한 줄)으로 대체한다. 없는 숫자를 지어내지 않는다(신뢰 붕괴). → §1-2 오프닝 참고.

### 1-2. 장점요약 오프닝 (공감 → 전환)

| 항목 | 내용 |
|---|---|
| **달성 과업** | 고객의 페인포인트를 먼저 말해 "내 얘기다"를 만들고, 그 감정을 제품으로 연결(전환) |
| **필수요소** | ① 공감형 질문(페인포인트, 병렬 3줄) ② 제품샷 ③ 강조 CTA 한 줄("그래서 이거예요"식 전환) |
| **톤** | 공감 → 전환. 질문은 부드럽게, CTA는 확신 있게 |
| **왜** | 사람은 "장점 나열"보다 **자기 문제를 정확히 짚어주는 쪽**에 마음을 연다. 질문 3줄을 같은 어미로 병렬하는 건 리듬으로 공감을 누적시키기 위함(→ 카피 규칙은 04 문서). 그 직후 CTA로 감정의 방향을 제품 쪽으로 튼다 |

### 1-3. 장점요약 번호형 (01~0N, 근거 축적)

| 항목 | 내용 |
|---|---|
| **달성 과업** | 스펙·인증·강점을 번호로 쌓아 "근거가 이만큼 많다"는 신뢰를 축적 |
| **필수요소** | ① 해시태그(라벨) ② 대형 번호(앵커) ③ 항목 구분자 ④ 항목 제목 ⑤ 하이라이트 문구(핵심 2줄) — 구분자의 조형 처리는 → 02 문서 |
| **톤** | 근거 제시. 차분하고 단정적, 정보를 정돈해 보여줌 |
| **왜** | 번호는 "빠짐없이 열거된다"는 완결감을 준다 → 각 항목이 하나의 증거로 읽힌다. 항목 사이를 무겁게 가르지 않고 구분자로만 끊어 "리스트의 호흡"으로 읽히게 한다(→ 구분자의 조형은 02 문서) |

> 비중 조절: 인증·특허·수상이 많을수록 번호를 늘려 지면을 키운다. 근거가 몇 개 없으면 억지로 번호형을 늘리지 말고 개성 섹션(§1-4)으로 통합한다.

### 1-4. 개성 섹션 (제품 차별점)

| 항목 | 내용 |
|---|---|
| **달성 과업** | 이 제품**만의** 강점을 설득·비교로 각인 |
| **필수요소** | 차별점 하나당 카드 한 장(카드마다 이미지 + 후킹 제목 + 설명 본문), 카드 사이 구분자 — 레이아웃·조형은 → 02 문서 |
| **카드화 원리** | 차별점을 **카드 단위로 격리해 각인**한다. 레이아웃 패러다임·화면비·반복 리듬은 → 02 문서 |
| **톤** | 설득·비교. "왜 이게 다른가"를 조목조목 |
| **왜** | 차별점은 **하나씩 격리해 보여줄 때** 각인된다 → 카드 단위로 분리. 각 이미지는 독립된 샘플처럼 카드에 담아 격리 제시한다(→ 이미지·조형 처리는 06·02 문서). 카드를 한 세트로 통일감 있게 배열하면 여러 강점이 "한 묶음"으로 읽힌다(→ 반복 리듬은 02 문서) |

### 1-5. 고객리뷰 (사회적 증거)

| 항목 | 내용 |
|---|---|
| **달성 과업** | "남들도 샀고 만족했다"는 사회적 증거로 마지막 망설임 제거 |
| **필수요소** | ① 댓글박스 여러 개(사진·닉네임·날짜·별점·본문, 본문 일부는 강조) ② 마무리 CTA("선택!"류) |
| **톤** | 신뢰. 진짜 후기의 생생함, 과장 없이 |
| **왜** | 브랜드가 하는 말보다 **제3자의 말**이 결정 직전에 가장 강하다. 댓글박스를 SNS 댓글 형태로 두는 건 "실제 사용자 목소리"라는 프레임을 주기 위함(→ 구체 조형은 02 문서). 닉네임 마스킹(OO**)은 진짜 후기라는 신호. 마무리 CTA로 마지막 확신을 준다(→ 강조 조형·타이포는 02·03 문서) |

> 비중 조절: 후기가 많으면 댓글박스 개수를 늘려 지면을 키운다(강한 자산에 몰아주기). 후기가 적으면 개수를 줄이되, 있는 후기는 **질 좋은 것 위주**로 크게 배치한다. 후기가 거의 없는 신제품이면 이 섹션을 생략하거나 최소화하고 §1-3 근거형에 무게를 옮긴다.

### 1-6. 안심 섹션 (법정 고지·구매조건 — 카테고리별 변주)

| 항목 | 내용 |
|---|---|

> **어느 구간을 히어로로 — 우선순위:**
> 1. **오프닝·클로징 — 무조건.** 첫인상·마지막 잔상은 아키타입 무관 히어로(정보형 페이지도 이 2곳은 권장). §1-1 소개메인·§1-2 오프닝·§1-7 브랜드 클로징이 이 타입의 특수형.
> 2. **(차순위) 직전 2섹션 연속 텍스트 고밀도** → 리듬 브레이크로 눈을 쉬게 하고 제품에 집중.
> 3. **욕구·정서 점화 역할**(먹고싶다/갖고싶다/선물하고싶다) 또는 **셀링포인트가 본질적으로 시각적**(질감·원물·세트·패키지·사용장면)인 구간.
> 4. **(추가 신호) 기획문서상 본문(설명)이 따라붙지 않는 단독 타이틀 구간** — 타이틀만 있고 설명 본문이 없다 = "여긴 글이 아니라 비주얼로 채우는 자리"라는 명시적 신호.
>
> ★ **오프닝·클로징 히어로도 텍스트 면적 12% 하한 예외 없음.** 순수 비주얼 오프닝/클로징이라도 카피를 **사진에 베이크하지 말고 라이브 텍스트 노드로** 올려 최소 12%(면적)를 채운다 — 사진에 구워 넣은 글자는 하한 판정에서 미집계되고 편집·선명도·접근성에서도 불리하다.

## 2. 섹션 순서 표준

순서는 **설득 시퀀스의 자연스러운 흐름**을 따른다: 붙잡고(첫인상) → 공감시키고 → 근거를 쌓고 → 차별화로 밀고 → 남의 말로 확증하고 → 안심시키고 → 정서로 남긴다.

| 순번 | 섹션 | 그 자리에 오는 이유 |
|---|---|---|
| 1 | 소개 메인/오프닝 히어로 | 컨셉+신뢰를 3초 안에 각인해 다음 스크롤로 잇는다 |
| 2 | 장점요약 오프닝(공감→전환) | 붙잡은 직후 "내 얘기"로 감정을 연다 |
| — | (중간제목 배너) | 이후 큰 화제 전환마다 삽입 — 고정 순번 없음, 리듬 브레이크 |
| 3 | 기능 간단 설명 / 장점요약 번호형(근거) | 열린 마음에 근거·기능을 빠르게 쌓아 논리 기반 마련 |
| 4 | 세부 구성표(구성물 여럿일 때) | "다 들어있다"는 인벤토리 완결감 |
| 5 | 기능 상세 설명 / 개성 섹션(차별화) | 근거 위에서 "그래서 이게 다르다"로 심화·밀어붙임 |
| 6 | 추천 대상(페르소나) | 차별화를 본 독자가 "내 얘기"로 자기 판별 |
| 7 | 고객리뷰(사회적 증거) | 결정 직전, 제3자의 말로 확증 |
| 8 | 가격/리워드 티어(있으면) | 확증된 독자를 옵션 선택으로 전환 |
| 9 | FAQ | 마지막 망설임(가격·사용법·환불) 선제 해소 |
| 10 | 안심 섹션(성분표/환불규정 등 카테고리별) | 법정 표기·구매조건 투명 공개 |
| 11 | 브랜드 공용 섹션(철학·팀소개·활용안내 등) | 구매 후에도 남는 브랜드 정서 — 공용 자산 복제 |
| — | (이벤트/공지) | 프로모션 기간에만 상단 또는 해당 위치에 삽입 |

> 이 순서는 **기본형**이다. 자산에 따라 강한 섹션을 앞당기거나 약한 섹션을 생략할 수 있다. 예: 후기가 압도적이면 리뷰를 조금 앞으로, 신제품이라 후기·인증이 얇으면 개성·서사형에 무게를 몰아준다. **신규 브랜드/펀딩 상품처럼 판매 실적이 없는 경우 "만든 사람이 누구인가"가 최전방 신뢰 카드가 되므로 팀/브랜드 소개를 오프닝 직후로 앞당기는 변주도 유효하다.** 순서를 바꾸는 근거는 항상 "어느 자산이 이 제품의 가장 강한 설득 카드인가"여야 한다.

## 2b. 펀딩(크라우드펀딩) 페이지 변주

커머스 상세페이지의 기본형에 **크라우드펀딩 플랫폼 관습**을 얹은 변주. 펀딩 플랫폼의 성공 사례·훅 유형·리워드 데이터는 브랜드 무관 재사용 가치가 있다.

**검증된 시퀀스(13섹션)**: 훅(공감형 슬로건) → 공감 심화(왜 다 실패했나) → **메이커 간증(팀 소개 앞당김 — 1인칭 "나는 이걸로 이미 앞서갔다" 서사)** → 솔루션 리빌 → 기능 간단 → 기능 상세 → 차별점 → 추천 대상 → 실증/신뢰(사실 서술만) → 리워드 티어 → 진행 일정 → FAQ → 안심/유의사항.

| 펀딩 특화 규칙 | 내용 |
|---|---|

## 3. 페이지 선택 결정 로직

### 3-1. 신제품 vs 스테디셀러 — 이건 "판단 기준"일 뿐 확정이 아니다

제품 성격은 섹션 구성의 **힌트**를 주지만, 최종 구성은 **인터뷰로 확정**한다(→ 00-interview 문서). 아래는 "이런 신호면 이쪽에 무게가 실리기 쉽다"는 경향일 뿐, 자동 규칙이 아니다.

| 신호 | 무게가 실리기 쉬운 섹션 | 왜 (경향의 근거) |
|---|---|---|
| 스테디셀러(후기 多, 판매 실적) | 소개 메인의 신뢰지표 강화 + 고객리뷰 크게 | 이미 쌓인 사회적 증거가 가장 강한 카드 → 그걸 전면에 |
| 신제품(후기 少, 실적 無) | 개성 섹션 / 서사형 / 근거형에 무게 + **팀/브랜드 소개 앞당김** | 남의 말이 없으니 **제품 자체의 논리·차별화 + 만든 사람의 신뢰**로 설득해야 함 |
| 인증·특허·수상 보유 | 장점요약 번호형 확장 | 객관적 근거 자산이 많음 → 번호로 축적 |
| 경쟁/기존방식 대비 우위가 핵심 | 서사형(문제→원인→해결) + 비포/애프터 비교 | 반박형 설득이 스펙 나열보다 강함 |
| 기능이 많은 제품(디지털·가전·세트) | 기능 간단·상세 2단계 구조 | 한 번에 다 설명하면 아무것도 안 읽힘 → 이중 깔때기 |
| 구성물이 여러 개(세트·템플릿·구독) | 세부 구성표 필수 추가 | "뭐가 포함되나"가 구매 결정 관문 |
| 식품 | 성분표(안심 섹션) 필수 추가 | 투명성이 식품 구매의 마지막 관문 |
| 디지털/무형 상품 | 환불규정 경고 우선 + 고시(안심 섹션) | 즉시 다운로드 상품의 환불 분쟁 예방이 관문 |
| 펀딩·프로모션형 | 가격/리워드 티어 + 이벤트 섹션 | 옵션 비교·가격 앵커링·기간 한정이 전환 축 |
| 감성·세계관이 브랜드 강점 | 철학 섹션 비중 ↑(단, 복제) | 정서가 재구매를 만드는 카테고리 |

### 3-2. 결정 절차 (원리)

1. **역할 리스트업**: 이 제품에 필요한 심리 과업을 먼저 나열한다(첫인상은 항상 필요, 나머지는 제품별로).
2. **자산 매칭**: 각 과업을 뒷받침할 보유자산이 있는지 확인한다(후기 몇 개? 인증 있나? 성분 데이터 있나?).
3. **비중 배분**: 자산이 강한 과업에 지면을 몰아주고, 자산이 없는 과업은 축소·생략하거나 대체 섹션으로 우회한다.
4. **인터뷰로 확정**: 신제품/스테디셀러 신호는 3-1처럼 **가설**만 세우고, 실제 구성·순서·비중은 인터뷰에서 사용자와 확정한다. **경향을 규칙으로 굳혀 임의 확정하지 않는다.**

> 왜 인터뷰가 최종인가: 같은 "신제품"이라도 어떤 건 근거형이, 어떤 건 서사형이 맞다. 제품의 **진짜 강점이 어디 있는지**는 데이터 신호만으로 못 정하고, 사용자가 가진 맥락(왜 만들었나, 무엇을 자랑하고 싶나)이 있어야 정확해진다. 신호는 대화를 여는 재료이지 결론이 아니다.
