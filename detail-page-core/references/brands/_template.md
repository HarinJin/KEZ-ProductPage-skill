# 브랜드 색·서체 프로파일 — <브랜드이름> (템플릿)

> 이 문서는 **한 브랜드의 실제 값(HEX·서체·규격·자산)**만 담는다. 패턴(어느 자리에 어떤 채도/명도/색조·위계를 쓰는가)은 `design-dna/*`. 이 문서는 그 패턴의 슬롯에 들어가는 값이다.
> **사용법**: 이 파일을 `brands/<브랜드이름>.md`로 복사해 아래 빈 슬롯을 채운다. 채우기 전 아래 "채우는 절차"를 따른다.

## 채우는 절차 (제작 착수 전 1회)

`design-dna/01-color.md` §5(새 브랜드 색 팔레트 도출) 절차에 따라:

1. **기존 페이지 실측** — 브랜드에 기존 상세페이지가 있으면 `get_design_context`로 대표 프레임 여러 장의 색·서체·규격을 실측한다. 근접값(스포이드 오차·미토큰화)은 **가장 많이 쓰인 값 하나로 정규화**하고, 죽은 스타일(사용처 없는 색)은 올리지 않는다(→ `01-color` §5).
2. **세대 정리** — 원본에 구버전·신버전이 섞여 있으면 신버전을 정본으로, 구버전은 아래 "레거시" 절에 기록만 남긴다.
3. **유저 확정** — 실측만으로 안 정해지는 값(브랜드 대표색·톤·서체 방향·미정 슬롯)은 **유저에게 확인**해 확정한 뒤 기록한다. **패턴만으로 HEX·폰트를 지어내지 않는다.**
4. **값 기록** — 아래 표에 역할→값으로 채운다. 미정 슬롯은 `(미정)` + `⬜`로 표기하고, 그 값이 필요한 섹션 제작은 확정 전까지 보류한다.
5. **확정 이력 남기기** — 무엇을 언제 유저 확정했는지, 무엇을 정규화·제거했는지 맨 아래 "확정 이력"에 기록해 재작업을 막는다.

> HEX는 예시가 아니라 **이 브랜드의 실측/확정 값**만 넣는다. 아래 표의 값 칸은 전부 비어 있다.
