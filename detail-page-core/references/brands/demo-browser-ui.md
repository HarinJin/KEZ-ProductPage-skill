# 브랜드 프로파일: demo-browser-ui (데모 — 브라우저 UI 톤)

> 이 스킬 자체의 깃허브 README 데모용 프로파일. 유저 제공 스크린샷(2026-07-16)에서 추출한 값 — **추출 추정치**이며 교정 가능. 실브랜드 아님. (전체 슬롯 구조의 예시가 아니라 톤 축약본 — 슬롯 구조는 `_template.md`가 정본.)

## 톤 선언

- **톤**: 개발자 도구/브라우저 UI — 정돈·미니멀·기술적 신뢰. 팝·스티커 톤 아님 → 강조는 박스 없는 대형 텍스트보다는 **얇은 보더 pill + 포인트 컬러**로 절제되게.
- **모티프**: 브라우저 창 크롬(신호등 3점 + URL 바), 코드/터미널 감성의 mono 라벨.

## 색 토큰 (이미지 추출 추정치)

| 토큰 | HEX | 역할 |
|---|---|---|
| INK | `#2D2D2D` | 짙은 회색 — 본문 텍스트, 다크 섹션 배경, 크롬 |
| CHROME | `#3A3A3A` | 브라우저 바·다크 카드 |
| ORANGE | `#ED7132` | 포인트 — pill, 넘버, 강조어, 액센트 라인 |
| BG_LIGHT | `#F5F4F2` | 밝은 섹션 배경(회백) |
| WHITE | `#FFFFFF` | 카드·다크 위 텍스트 |
| LINE | `#E3E1DD` | 보더·구분선 |
| TRAFFIC | `#FF5F57` `#FEBC2E` `#28C840` | 브라우저 신호등(macOS 표준) — 크롬 모티프 전용 |

- 색 규칙: 밝은 배경 위 강조 = ORANGE(채도). 다크(INK) 배경 위 텍스트 = WHITE/회백(명도). ORANGE는 페이지당 앵커 요소에만 — 남발 금지.

## 서체 역할 (가용성은 착수 시 listAvailableFontsAsync로 검증)

| 역할 | 1순위 | 대체 체인 |
|---|---|---|
| 기본(KO+EN 본문·제목) | Pretendard | → Noto Sans KR → 검증 결과 보고 후 결정 |
| 정보(mono — URL·코드 라벨·규칙 스트립) | JetBrains Mono | → Roboto Mono → Space Mono → 기본 서체 Medium |
| 감성 | 없음 (단일+mono 2역할) | — |

## 규격 (860 캔버스 기준)

| 규격 | 값 | 비고 |
|---|---|---|
| 아트보드 가로폭 | 860px | 기본값 |
| 텍스트 컨테이너 최대폭 | ≤725px (84%) | 하드 리밋. full bleed는 순수 시각물만 |
| 본문 최소 | ≥39px (4.5%) | KO 본문 40 관용 |
| 캡션·EN 병기 최소 | ≥26px (3%) | EN 보조 병기 28 관용 |
| 디스플레이/타이틀 | ~72–90px | 본문의 약 2배↑ |
| pill 라벨 | 28–30 Bold, **hug width** | 좌우 패딩 24–32 |
| 카드 라운딩 | 16px | 브라우저 창 목업은 12px |
| 브라우저 크롬 바 높이 | 56px | 신호등 지름 16, 간격 10 |
| KO/EN 병기 정책 | KO 주(본문급) + EN 보조(캡션급, 회색) | 같은 블록 안 위계 점프로 |
