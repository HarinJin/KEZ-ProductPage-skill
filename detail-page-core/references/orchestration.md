# 병렬 섹션 제작 — 감독(오케스트레이션) 절차

> 전체 신규 페이지를 서브에이전트 여러 명으로 병렬 제작할 때의 절차와 **복붙용 프롬프트 템플릿**. 다수 섹션·다수 빌더·복수 웨이브 규모에서 검증됨.
> 목적: 감독 모델이 무엇이든 이 템플릿대로 지시하면 빌더의 시행착오(함정 재발견·재작업) 토큰을 최소화한다.

## 0. 원칙

- 스켈레톤 승인 후 착수. **섹션 = 독립 프레임 = 빌더 1명**, 웨이브당 4~5명 병렬.
- 빌더는 **자기 프레임 안에서만** 작업. 다른 프레임 수정·이동 절대 금지. 높이가 바뀌면 자기 프레임만 조정하고 최종 높이를 보고.
- 프레임 겹침은 빌더가 아니라 **감독이 웨이브 종료마다 재배치(§3)**로 해소한다.
- **감독이 모든 섹션을 스크린샷으로 직접 검수**한다(빌더 자가검증 보고만 믿지 않는다). 긴 프레임 검수법은 `troubleshooting.md` T12.

## 1. 착수 전 감독이 1회 준비 (빌더 반복 낭비 방지)

1. **폰트 전수 검증**: 브랜드 프로파일의 모든 폰트(기본+손글씨+정보체)를 `listAvailableFontsAsync()`로 1회 확인 → **가용/미보유+대체 규칙을 모든 빌더 프롬프트에 명기**. 빌더마다 재확인시키면 그만큼 낭비 (→ `troubleshooting.md` T7).
2. **공용 조형 값 고정**: 배지 pill·창 목업·라운딩·(브랜드가 시그니처로 선언한 경우만) 섀도 등 섹션 간 공통 조형의 규격(브랜드 프로파일 값)을 프롬프트에 못박아 섹션 간 일관성을 확보한다. "파일 내 기존 섹션(N번) 실측 참고"도 유효.
3. **카피 소스 고정**: 기획서(카피 확정본) 경로와 "원문 그대로, 창작·의역 금지"를 명기. 절대 금지 3종(법정 고지문·리뷰·미확정 카피)과 placeholder 정책(금액·날짜·수량 미확정 값은 빈 슬롯)도 프롬프트에 포함.

## 2. 빌더 프롬프트 템플릿 (복붙 후 [ ] 채우기)

```
너는 Figma 상세페이지 빌더다. [제품명]의 [NX 섹션명] 섹션을 세부 제작하라.

## 대상
- fileKey [키], 프레임 [노드ID] ("[프레임명]", x=0 y=[y], 860×[h]).
  **이 프레임 안에서만 작업. 다른 프레임 절대 수정·이동 금지.**
  높이 조정 시 자기 프레임만 바꾸고 최종 높이를 보고하라.
- 스켈레톤 placeholder(섹션라벨·핵심카피·이미지슬롯 등)는 세부 제작으로 전부 대체하라(잔존 = 결함).

## 먼저 읽을 것
1. /figma-use 스킬 (없으면 skill://figma/figma-use/SKILL.md 리소스)
2. 이 스킬의 figma-mcp.md, build-recipes.md, troubleshooting.md(에러 시 증상 색인)
3. [기획서 경로] — [해당 섹션 절] (+확정 내역 절)
4. brands/[이름].md — 색·서체·규격 값

## 제작 지시
- 카피는 기획서 확정본 **원문 그대로** (창작·의역 금지). 미확정 값([항목])은 placeholder 슬롯으로 비움.
- [섹션 고유 지시: 조형/공용 자산 실측 재구현(원본 파일·노드ID)/이미지 슬롯·촬영 라벨]
- **폰트**: [전수 검증 결과 — 가용 목록 + 미보유 목록과 대체 규칙]. 미보유 폰트 사용 금지.
- **타이포 절대 규칙**: 본문 ≥ 캔버스폭의 4.5%(860폭→39px, 1000폭→45px), 캡션 ≥ 3%(860폭→26px), 마켓플레이스 게재용 본문 4.7%↑, 인접 블록 크기+굵기 점프(템포). 색: [브랜드 토큰]. 레거시 값 금지.
- **레이아웃 절대 규칙**: 텍스트가 든 컨테이너 최대폭 = 캔버스폭의 84%(860폭→725px) 초과 금지 — full bleed는 순수 시각물만. 텍스트박스 사이징은 사전 판정: 타이틀·강조 pill=hug width / 본문(2문장 이상)=fixed width+hug height (→ `design-dna/02` §1·§7).
- 구조: 모든 노드 한글 역할명, 중첩 오토레이아웃, 정렬 래퍼 fills 비움.

## 완료 기준
get_metadata + get_screenshot 자가 검증(겹침·잘림·최소크기·네이밍·placeholder 잔존) 후
**반드시 최종 텍스트로 보고**: 최종 높이, 노드 구성, 폰트 이슈, 스크린샷 소견.
보고 없이 종료 금지.
```

> "반드시 보고"를 강조하지 않으면 빌더가 무보고 idle로 빠진다(강조 후 전원 보고로 개선됨). 무보고 idle이 와도 당황하지 말 것 — 대개 **작업은 완료**돼 있으니 프레임을 직접 스크린샷으로 확인한다.

## 3. 재배치(re-flow) 스니펫 — 웨이브 종료마다 감독이 실행

```js
const order = ['3:2','3:7', /* ...섹션 프레임 ID 순서대로 */];
const GAP = 40;
let y = 0;
const frames = [], mutated = [];
for (const id of order) {
  const node = await figma.getNodeByIdAsync(id);
  if (!node) continue;
  const h = Math.round(node.height);
  if (node.height !== h) node.resize(node.width, h);   // 소수점 높이 정수화 (T5)
  if (node.y !== y) { node.y = y; mutated.push(id); }
  if (node.x !== 0) node.x = 0;
  frames.push({id, name: node.name, y, height: h});
  y += h + GAP;
}
return { frames, mutatedIds: mutated, totalHeight: y - GAP };
```

## 4. 웨이브 검수 체크리스트 (감독, 섹션마다)

- [ ] 카피가 기획서 원문과 일치(창작·의역·오탈자 없음), placeholder 슬롯이 비어 있음
- [ ] 최소 텍스트 크기(본문 ≥ 캔버스폭 4.5% / 캡션 ≥3% — 860 기준 39/26px) + 인접 블록 템포(크기·굵기 점프)
- [ ] 텍스트 컨테이너 최대폭 ≤ 캔버스폭 84%(860 기준 725px, full bleed는 시각물만) + 타이틀 pill=hug width·본문=fixed width+hug height
- [ ] 스켈레톤 작업용 라벨·placeholder **잔존 없음**
- [ ] 촬영/이미지 슬롯에 촬영 번호·지시 라벨 명기
- [ ] 색 규칙 준수(경고색은 경고 전용, 컴포넌트 로컬 색은 해당 컴포넌트 한정, 레거시 값 없음)
- [ ] 겹침·잘림·의도치 않은 흰 박스 없음(스크린샷), 프레임 간 좌표는 재배치 후 확인(메타데이터)
- [ ] 전 웨이브 종료 후: 최종 재배치 → 전체 스트립 스크린샷으로 밴드 리듬·연속성 확인
