# 트러블슈팅 색인 — Figma MCP·Plugin API (증상 → 해법)

> **에러·이상 렌더가 나면 먼저 이 색인에서 증상을 찾는다.** 없으면 figma-use 스킬 gotchas 확인. 새로 해결한 함정은 이 문서에 추가한다(증상/원인/해법).
> use_figma는 **원자적** — 실패한 스크립트는 아무것도 반영 안 됨(고아 노드 없음). 즉시 재시도 금지, 에러 메시지 읽고 수정 후 재시도.

## T1. textAutoResize 순서 (실측 검증)
- **원인**: `textAutoResize='HEIGHT'` 설정 **후** `resize(w, 임의높이)`를 호출하면 높이가 그 임의값에 고정된 채 렌더까지 깨진다(메타데이터 캐시 문제 아님 — 스크린샷에서도 잘림).
- **해법**: `textAutoResize='NONE'` → `resize(w, 아무값)` → `textAutoResize='HEIGHT'` 순서로 재설정하면 줄바꿈 높이가 정상 재계산된다.

## T2. 깊은 중첩 텍스트 줄바꿈 미계산 (실측 검증 — 스크린샷 5회 재확인 끝에 특정)
- **원인**: `섹션→리스트→row→본문`처럼 오토레이아웃 3~4단 중첩에서 grandchild 텍스트에 `FILL`+`textAutoResize='HEIGHT'`를 주면, **같은 스크립트 실행 내**에서는 부모의 hug 높이가 재계산되지 않아 텍스트가 잘려 보인다. 최상위에서 1홉 자식은 정상.
- **해법**: 깊이 중첩된 텍스트는 자동 줄바꿈에 의존하지 말 것 — `\n` 수동 개행 + 각 프레임 높이를 명시적 FIXED 값으로 지정.

## T3. hug 재계산 강제 (실측 검증)
- **원인**: 자식을 여럿 append한 뒤에도 hug(primaryAxisSizingMode=AUTO) 컨테이너가 height=1 등으로 남아 뒤 형제와 겹침.
- **해법**: ① 자식 실측 높이를 합산해 FIXED로 명시, 또는 ② sizing mode를 FIXED→AUTO로 **토글**해 강제 재계산.

## T4. query() 한글 셀렉터 미지원 (재확인됨)
- **원인**: `node.query('TEXT[name=섹션라벨]')`처럼 속성값에 한글이 들어가면 `Invalid selector: unexpected character (0xec)`.
- **해법**: 한글 역할명이 이 스킬의 표준이므로 **쿼리는 항상 findAll로**: `frame.findAll(n => n.type==='TEXT' && n.name==='섹션라벨')`.

## T5. 소수점 높이 전파 (실측 검증)
- **원인**: hug 계산 결과 프레임 높이가 소수(예 3167.47)가 되면, 그 아래로 재적층하는 모든 프레임 y가 소수로 밀린다(렌더 블러 위험).
- **해법**: 재배치 전 `node.resize(w, Math.round(node.height))`로 정수화 → 이후 y 적층도 정수 유지.

## T6. 크로스파일 자산 재구현 (검증됨)
- **원인**: 다른 파일의 컴포넌트는 이 파일로 직접 복제(clone)할 수 없다(미게시 로컬 컴포넌트).
- **해법**: 원본 노드를 `get_design_context`/`get_metadata`로 **실측**(치수·라운딩·보더·섀도·폰트)한 뒤 동일 조형으로 재구현. 눈대중 금지. **주의**: 원본이 레거시 값(구세대 색·폰트)일 수 있음 — 조형 문법만 가져오고 색·폰트는 브랜드 프로파일의 현행 값으로 대입(실측 사례: 공용 폼 컴포넌트가 레거시 색·폰트였음).

## T7. 폰트 로딩·가용성
- `loadFontAsync` 완료 후에만 텍스트 조작. 기존 텍스트는 `getStyledTextSegments(['fontName'])`로 현재 폰트를 읽어 로드(하드코딩 금지).
- **착수(스켈레톤) 단계에서 브랜드 프로파일의 모든 폰트를 `listAvailableFontsAsync()`로 1회 전수 확인**하고 결과를 이후 전 작업(빌더 프롬프트 포함)에 전파한다. 미보유 발견이 작업자마다 반복되는 것이 최대 토큰 낭비(실측 사례: 손글씨 폰트 미보유가 여러 섹션에서 각각 재발견됨).
- 미보유 시: 브랜드 프로파일에 대체 규칙이 있으면 따르고(예: 손글씨 스탬프→기본 서체 Bold+회전, 인용구→따옴표 글리프+위계), 없으면 유저에게 업로드 요청. 임의 유사 폰트 대체 금지.

## T8. createAutoLayout 기본 흰배경 (반복 발생)
- **증상**: 정렬용 래퍼의 기본 흰 fill이 콘텐츠 위로 겹쳐 보임(텍스트 대비 뭉갬).
- **해법**: 배경이 필요 없는 래퍼는 생성 직후 `node.fills = []`.

## T9. union fill 리셋 (검증됨)
- **증상**: `figma.union()` 결과가 회색. **해법**: union 직후 `shape.fills = [...]` 재지정.

## T10. 부모 resize 자식 왜곡 (관측 8.6배)
- **증상**: 자식 append 뒤 `container.resize()` 호출 시 자식이 SCALE 제약으로 확대.
- **해법**: 컨테이너 최종 크기는 생성 직후·append 전에 확정. 필요시 자식 `constraints={horizontal:'MIN',vertical:'MIN'}`. 복잡한 마스크 자산은 `resize` 대신 `rescale()`.

## T11. 장식 vector 비종속
- **증상**: 복제한 밑줄/강조 vector가 새 카피 길이와 어긋남.
- **해법**: 새 텍스트의 실제 렌더 폭 확인 후 x/y/width 재계산. 안 맞으면 빼고 유저 확인.

## T12. 긴 프레임 검수법 (검증 파이프라인)
- **증상**: 세로 3000px+ 프레임을 기본 maxDimension으로 찍으면 폭이 수백px 이하로 눌려 텍스트 판독 불가.
- **해법**: `get_screenshot(maxDimension=원본높이)`로 원본 해상도 확보(`contentsOnly:true` 권장) → curl 다운로드 → PIL로 세로 2분할(필요시 100px 오버랩) → Read로 각 조각 검수.

## T13. 렌더 아티팩트 (실측 검증)
- **증상**: 전체 캔버스/섹션 스크린샷에 실제 노드가 아닌 색 밴드·비정상 폭이 보임(겹침 콘텐츠의 렌더 잔상).
- **해법**: 놀라지 말고 `get_metadata`로 좌표·폭을 확인해 실노드 여부부터 판정. 실노드가 아니면 `contentsOnly:true`로 재촬영.

## T14. 크롭 이미지 사전 검수 (실측 검증 — 원본 텍스트 박힘 사고)
- **증상**: 기존 캡처에서 크롭한 이미지에 원본의 텍스트·손글씨가 함께 박힌 채 업로드됨(y 범위 오판).
- **해법**: 크롭 결과를 업로드 **전에 반드시 Read로 열어** 잔존 텍스트·장식 유무를 눈으로 확인한다.
